

## Plan: Import Client Data and Create Relational Structure

### Data Analysis
The spreadsheet contains ~3,978 rows with: CodSupervisor, Supervisor, CodRepresentante, Representantes, Rede, Codigo do Cliente, Cliente. These have hierarchical relationships: Supervisor → Representante → Cliente (with Rede as a client attribute).

### Database Changes

**1. Create `supervisores` table**
- `id` (uuid, PK), `codigo` (integer, unique), `nome` (text)
- Populated with distinct supervisor entries (~6-8 unique: Vanessa, Rogerio, Dayana, Barbara, Patricia, Alice)

**2. Create `representantes` table**
- `id` (uuid, PK), `codigo` (integer, unique), `nome` (text)
- Populated with distinct representante entries (~30-40 unique)

**3. Create `supervisor_representante` junction table**
- Links supervisors to their representantes (a representante can belong to multiple supervisors)
- `supervisor_id` (FK), `representante_id` (FK)

**4. Create `redes` table**
- `id` (uuid, PK), `nome` (text, unique)
- Populated with distinct Rede values (excluding "SEM REDE")

**5. Update `clientes` table**
- Add columns: `codigo` (integer), `representante_id` (FK to representantes), `rede_id` (FK to redes, nullable)
- Populate with all ~3,978 client entries from the spreadsheet

**6. RLS policies** — All new tables get SELECT for authenticated + ALL for admin/gestor.

### Frontend Changes (NovoChamado.tsx)

- **Supervisor** dropdown: load from `supervisores` table instead of mockUsers
- **Representantes** dropdown: filter by selected supervisor via junction table
- **Código do Cliente**: filter by selected representante
- **Clientes**: auto-populate when código is selected (and vice-versa)
- **Rede**: auto-populate based on selected client's rede association

Cascading logic: Supervisor → filters Representantes → filters Clientes (código + nome) → auto-fills Rede.

### Data Import Strategy
Due to the large volume (~4,000 rows), the data will be inserted via multiple migration SQL statements:
1. First migration: create tables + insert supervisores and representantes (small datasets)
2. Second migration: insert all clientes with their relationships
3. The insert statements will be generated by extracting distinct values from the parsed spreadsheet data
